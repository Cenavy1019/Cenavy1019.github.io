meta标签的用法

论meta标签的部分用法

“meta除了定义编码格式，还有一些其他用法”、

1、定义文档优先默认选择的浏览器

<meta http-equiv="X-UA-Compatible"  content="IE=edge,chrome=1" />

2、定义文档支持的浏览器内核

<meta name="renderer" content="webkit|ie-comp|ie-stand" />

3、设置搜索关键词keywords

最好不超过12个关键词：
<meta name="keywords" content="袜子，裤子，衬衫，内裤，内衣，鞋子，皮包"

4、给title设置一段描述句

所增加的描述词最好不超过120个字。
<meta name="description" content="120 words" />

示范：

<!doctype html>
<html>
<head>
    <meta charset="utf-8" />
    <meta http-equiv="X-UA-Compatible"  content="IE=edge,chrome=1" />
    <meta name="renderer" content="webkit|ie-comp|ie-stand" />
    <meta name="keywords" content="衬衫，T恤衫，睡衣，裤子，男装，女装" />
    <title>衣装</title>
</head>
<body>
    <div class="left">
        <img src="#" alt="衣装" />
    </div>

        <p>
            <strong>衣装</strong>

浅谈自适应布局

这里我采用的是ul
图片自适应窗口大小靠的是li，所以一切最大最小的数值赋予li就够了，ul可以不用设置，唯独定义一个总宽度就够了。
图片自适应li大小。 width & height：100%；
当一排li总宽度>=ul设定的最大宽度时，无变化。
当只给ul一个最小宽度时，li内容只会自适应缩小，但不会自适应成行排列。
给li设置最小宽度时，当窗口缩小到它的极限时，内容会自动成行排列。
ul的最小宽度是控制视窗大小。低于最小则不动。
li的最小宽度是控制内容显示大小，同上。
li的总宽度要小于ul的最小宽度时候，才会被挤下去成行排列。

css3的单位

css3的单位

1.em
2.rem
3.max-width 最大宽度
4.max-height 最大高度
5.min-height 最小高度
6.min-width 最小宽度
7.vw 
8.vh

em和rem的区别

两者都是倍数单位，都可以设置小数，区别在于：
em是继承于它自己和它父级的字号而设置的；
rem是参照与html设置的字号而设置的，若设置的字号小于默认的字号，则参照默认字号大小设置。

max-width

定义元素的最大宽度。
该属性值会对元素的宽度设置一个最高限制。因此，元素可以比指定值窄，但不能比其宽。不允许指定负值。

max-height

定义元素的最大高度。
该属性值会对元素的宽度设置一个最高限制。因此，元素可以比指定值低，但不能比其高。不允许指定负值。

min-width

定义元素的最小宽度。
该属性值会对元素的宽度设置一个最小限制。因此，元素可以比指定值宽，但不能比其窄。不允许指定负值。

min-height

定义元素的最小高度。
该属性值会对元素的宽度设置一个最小限制。因此，元素可以比指定值高，但不能比其低。不允许指定负值。

vh

相对于视口的高度。视口被均分为100单位的vh。

vw

相对于视口的宽度。视口被均分为100单位的vh。

博客搭建

第一步 下载nodeJS

1.安装nodeJS：https://nodejs.org/en/
2.测试nodeJS    终端里输入node -v

第二步 安装hexo

1.运行npm install hexo -g 命令
2.运行root Artasdlkajs 命令
3.在sudo nam install hexo -g 然后输入自己的密码
4.测试hexo
5.创建目录进入，hexo init 是初始化hexo目录，hexo server 启动本地服务

第三步 github账号

一、配置sshkey
1.终端命令ssh-key gen 按三次回车
2.vim ~/.ssh/id_rsa.pub 复制ssh有用
3.github 设置→ssh栏里 粘贴刚才复制的ssh

二、配置github与本地机器验证
1.git config —global user name “请输入自己github的用户名” 
2.git config —global user name “请输入自己的github密码”

三、新建博客域名仓库
用户名.github.io

第四步 修改config.yml文件

1.找到deploy字段
2.在type后面加上 （要加空格，请去掉括号）git
3.repo： 复制粘贴域名仓库的http地址

第五步 发布自己的文档

1.nam install heao-deployer-git—save
2.终端输入hexo g 进行文件转html和css及js格式文件
3.终端输入hexo d 进行上传，看到done：git说明已经成功，打开自己的博客域名地址就能看到了。

PS 修改主题

1.找到自己喜欢的主题的域名，复制
2.终端里先进入主题目录
3.输入git clone http：//我是主题域名地址, 回车 下载主题
4.打开config.yml文件，找到themes，修改成新主题名字
5.跳转到blog目录下，输入hexo server启动服务
6.复制网址查看是否变动。
7.还可以自己修改背景图。
